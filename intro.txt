NEO SMART HOME - PROJECT DOCUMENTATION

==============================================

INTRODUCTION ABOUT PROJECT

Neo Smart Home is an IoT-based home automation system that allows users to control their home environment using multiple methods. Instead of relying only on a mobile app, this system lets you control lights, fans, and other appliances using your voice, hand gestures, and a web dashboard. The system also monitors room temperature, humidity, and motion to automatically respond to changing conditions.

The goal of this project is to make home automation more natural and accessible. You do not need to pick up your phone every time. You can simply speak a command, wave your hand, or let the system decide based on sensor readings.

This project is built using Python for the web backend, NodeMCU boards with DHT11 and PIR sensors for environmental monitoring, and an ESP32-CAM with laptop webcam for gesture recognition.

==============================================

WHAT IS IN THIS PROJECT

The project has five main components:

A. Environmental Monitoring System

Two NodeMCU boards placed in living room and bedroom

Each NodeMCU has a DHT11 sensor for temperature and humidity

Each NodeMCU has a PIR sensor for motion detection

Sensors send data every 2 seconds to the central Python server

B. Gesture Control System

ESP32-CAM placed in hallway for room-level gesture control

Laptop webcam for desk-level gesture control

Recognizes hand gestures like palm, fist, swipe left, swipe right, point up, point down

Each gesture triggers a specific action like turning on light or changing fan speed

C. Voice Assistant

Offline voice recognition (no internet required after setup)

Wake word "Hey Neo" activates the assistant

Understands commands like "turn on living room light" or "what is the temperature"

Responds with text-to-speech voice output

D. Web Dashboard

Built with Python Flask

Shows live temperature and humidity graphs

Displays motion detection history with timestamps

Provides manual buttons to override any appliance

Shows last recognized gesture and voice command

E. Relay Control System

5V relay modules connected to each NodeMCU

Controls lights, fans, and other appliances

Can be triggered by voice, gesture, sensor readings, or manual button

THINKING AND LOGIC USED IN THIS PROJECT

The core thinking behind this project is to solve real problems with practical solutions.

Problem 1: What if one device fails?
Solution: I used separate devices for sensing and vision. NodeMCU handles temperature and motion. ESP32-CAM handles gestures. If the camera stops working, temperature monitoring continues. If NodeMCU fails, gesture control still works for other devices. No single point of failure.

Problem 2: What if internet goes down?
Solution: Everything works offline. Voice recognition uses local libraries. Gesture recognition runs on the device itself. The web dashboard runs on local network. Only optional features like email alerts need internet.

Problem 3: How to make it fast?
Solution: I used MQTT protocol for sensor data because it is lightweight. I used WebSockets for real-time dashboard updates. Temperature updates appear on web page without refreshing. Motion alerts trigger in under 100 milliseconds.

Problem 4: How to make it work for everyone?
Solution: Multiple input methods. Elderly family members can use voice. Tech-savvy users can use gestures. Anyone can fall back to the web dashboard buttons. The system does not force one way of control.

Problem 5: How to handle false motion alerts?
Solution: The PIR sensor has adjustable sensitivity. I set it to ignore small movements. The system also checks for multiple motion detections within 10 seconds before sending an alert. This reduces false alarms from pets or curtains moving.

Problem 6: How to make gesture recognition reliable?
Solution: I set a confidence threshold of 70 percent. If the camera is not sure about a gesture, it does nothing. The system also logs all unrecognized gestures so I can retrain the model later. Good lighting conditions are important, so the ESP32-CAM has a built-in LED that turns on in low light.

Problem 7: How to avoid voice assistant confusion?
Solution: The assistant only listens after hearing the wake word "Hey Neo". It then records the next 3 seconds of audio. If the command is unclear, it asks "Did you mean light or fan?" on the web dashboard. This prevents accidental actions.

WORKING METHOD - STEP BY STEP

Here is how the entire system works from start to finish.

Step A: Power On and Initialization

All NodeMCU boards connect to WiFi

ESP32-CAM connects to WiFi

Python Flask server starts on the laptop

MQTT broker starts (or serial communication as fallback)

All devices register themselves with the server

Step B: Continuous Monitoring (Every 2 Seconds)

NodeMCU reads DHT11 sensor for temperature and humidity

NodeMCU checks PIR sensor for motion

Data is sent to Python server via MQTT

Server stores data in SQLite database

Server pushes update to all connected web browsers via SocketIO

Step C: Gesture Control Flow

ESP32-CAM captures video frames continuously

MediaPipe detects hand landmarks in each frame

Gesture classifier decides what hand pose is shown

If confidence is above 70 percent, command is sent to server

Server identifies which appliance to control

Server sends command to appropriate NodeMCU via MQTT

NodeMCU toggles relay to turn appliance ON or OFF

Web dashboard shows "Gesture Detected: Swipe Right" message

Step D: Voice Control Flow

User says "Hey Neo"

Microphone captures audio

Wake word detection triggers recording

Speech recognition converts speech to text

Text is parsed for keywords like "light", "fan", "on", "off"

Server matches command to appliance

Command sent to NodeMCU via MQTT

Text-to-speech says "Living room light turned on"

Step E: Automatic Triggers (Rules Engine)

If temperature exceeds 30 degrees Celsius, fan turns on automatically

If temperature drops below 18 degrees, fan turns off

If motion detected between 11 PM and 6 AM in bedroom, buzzer sounds

If motion detected in living room during day, web dashboard shows alert only (no buzzer)

Step F: Web Dashboard Interaction

User opens browser and goes to local IP address port 5000

Page loads current temperature, humidity, and motion status

Graphs show last 24 hours of data

User can click buttons to manually turn appliances ON or OFF

User can view motion log with timestamps

User can see voice command history

All actions update in real time without page refresh

DEVICE SUMMARY

NodeMCU ESP8266 - Two units
Used in living room and bedroom
Reads temperature, humidity, and motion
Controls relays for lights and fans
Connected via WiFi using MQTT protocol

DHT11 Sensor - Two units
Connected to each NodeMCU
Measures temperature from 0 to 50 degrees Celsius
Measures humidity from 20 to 80 percent
Accuracy is plus or minus 2 degrees

PIR Sensor HC-SR501 - Two units
Connected to each NodeMCU
Detects motion up to 7 meters away
Adjustable delay and sensitivity
Triggers on human body heat

ESP32-CAM - One unit
Placed in hallway
Captures video for gesture recognition
Has built-in OV2640 camera
Runs gesture model on device

Laptop Webcam - Built-in or USB
Used for desk-level gesture control
Higher resolution than ESP32-CAM
Works with MediaPipe library on laptop

Relay Modules - Four units
Two in living room, two in bedroom
5V operation controlled by NodeMCU pins
Switches lights, fans, and other appliances
Opto-isolated to protect NodeMCU from voltage spikes

==============================================

WHY THESE SPECIFIC DEVICES WERE CHOSEN

NodeMCU instead of Arduino Uno

Built-in WiFi means no extra shield needed

Cheaper than Arduino with WiFi shield

Can be programmed with Arduino IDE or MicroPython

DHT11 instead of DHT22

Sufficient accuracy for home automation

Much cheaper than DHT22

Response time is fast enough for room monitoring

PIR instead of ultrasonic sensor

Detects human body specifically

Ultrasonic sensors detect any movement including curtains

Lower power consumption

ESP32-CAM instead of Raspberry Pi with camera

Much cheaper (around 10 dollars vs 50 dollars)

Built-in processor runs TensorFlow Lite

Smaller form factor for hallway mounting

Laptop webcam instead of external camera

Zero additional cost

Already positioned for desk use

Higher resolution for better gesture detection

==============================================

COMMUNICATION BETWEEN DEVICES

NodeMCU to Python Server

Uses MQTT protocol by default

Topics: living/temp, living/motion, bedroom/temp, bedroom/motion

If MQTT broker is down, falls back to serial over USB

ESP32-CAM to Python Server

Uses WebSocket for real-time bidirectional communication

Video stream uses HTTP MJPEG over port 8080

Gesture events sent as JSON messages

Python Server to Web Browser

Uses SocketIO for push notifications

REST API for manual button clicks and data fetching

No page refresh required for updates

EXAMPLE USE CASE SCENARIO

Morning 8 AM:
User enters living room. PIR sensor detects motion. Living room light turns on automatically. Web dashboard shows "Good Morning" message.

User says "Hey Neo, what is the temperature?" Voice assistant replies "Temperature is 24 degrees. Humidity is 55 percent."

User waves hand with open palm. ESP32-CAM detects palm gesture. Fan turns on at medium speed.

Afternoon 2 PM:
Temperature rises to 32 degrees. Automatic rule triggers. Fan speed increases to maximum. User receives web dashboard notification.

Evening 8 PM:
User sits at desk. Laptop webcam detects user presence. Desk lamp turns on automatically.

User swipes left in front of laptop camera. Music player skips to next song.

Night 11 PM:
User says "Hey Neo, good night." All lights turn off. Bedroom fan turns on at low speed. Security mode activates.

Night 2 AM:
PIR sensor detects motion in bedroom. System checks time and sees it is after 11 PM. Buzzer sounds briefly. Web dashboard logs "Motion detected at 2:15 AM". User can check log in morning.

CONCLUSION

Neo Smart Home successfully integrates five different control methods into one unified system. The project proves that home automation does not need to rely on expensive commercial products or cloud services. Using affordable hardware like NodeMCU and ESP32-CAM with open source software like Python Flask and MediaPipe, a fully functional smart home system can be built for under 50 dollars.

The multi-modal approach ensures that users of all ages and technical abilities can interact with their home naturally. Voice for convenience, gesture for speed, sensors for automation, and web dashboard for reliability.

Future improvements can include face recognition for personalized scenes, mobile app for remote access, and energy monitoring to track power consumption.
